# OFC Solver

Open-Face Chinese Poker (Pineapple + Fantasyland) AI solver, evaluator,
and analysis server.

## What's in this repo

| Path | Purpose |
|---|---|
| `engine/` | Core game model: cards, deck, evaluator, royalties, scoring, fantasy rules |
| `state/` | Mutable game state, board, actions |
| `ai/` | Policies: random, heuristic, Monte-Carlo, fast-opponent (for rollouts) |
| `fantasy/` | Exact fantasy-hand solver |
| `tables/` | Precomputed lookup tables (opening book, foul prob, fantasy EV, …) |
| `simulation/` | Self-play harness and trace collectors |
| `scripts/` | Table builders, benchmarks, profilers |
| `ui/` | Local HTTP analysis server + minimal web UI |
| `tests/` | Pytest suite (200+ tests) |
| `docs/API.md` | Full HTTP API reference for `ui/server.py` |
| `ofc_ai.py` | Public Python API (`OFCAI` class) for embedding into other apps |

## Quick start

```bash
# Install (editable) — Python 3.10+
pip install -e .

# Run the test suite
python -m pytest -q

# Start the local analysis server
python -m ui.server --host 127.0.0.1 --port 8000
# then open http://127.0.0.1:8000/ in a browser

# Programmatic use
python -c "
from ofc_ai import OFCAI
ai = OFCAI(tables_dir='artifacts/run_100k_20260508_103044')
print(ai.recommend({
    'street': 1, 'to_act': 0,
    'players': [
        {'fantasy_tier': 0, 'board': {'top':[],'middle':[],'bottom':[],'discards':[]}, 'pending': ['As','Kd','Qh','Jc','Tc']},
        {'fantasy_tier': 0, 'board': {'top':[],'middle':[],'bottom':[],'discards':[]}, 'pending': []},
    ],
}))
"
```

## Precomputed tables

Tables (`*.pkl` under `artifacts/`) are **not** tracked in git — they are
large (up to ~100 MiB each) and regenerated by build scripts. Manage
them manually (release attachments, object store, rsync, etc.).

To rebuild from scratch:

```bash
# Canonical opening book (152,646 entries, ~14 h on 16 workers)
python -m scripts.build_full_opening_book \
    --workers 16 --n-rollouts 60 --top-k 5 \
    --out artifacts/opening_book_canonical

# Self-play tables (foul prob, policy prior, fantasy arrangement, …)
python -m scripts.build_all_tables \
    --n-games 100000 --n-workers 16 \
    --p0 mc --p1 mc --fantasy-rate 0.20 \
    --out artifacts/run_100k
```

Point `OFCAI(tables_dir=...)` or `ui.server --tables-dir ...` at the
output directory.

## Project layout notes

- Python 3.10+ required (uses PEP 604 union types).
- No third-party runtime deps — `pip install -e .` installs only this
  package.
- Multiprocessing uses the `"spawn"` start method (forks deadlock against
  the analyzer's internal thread state).
- The UI server is reentrant-safe: concurrent `/api/analyze` requests
  share a single rollout `Pool` and are gated by a configurable
  semaphore (`--max-concurrent`).
- For an HTTP-only quick reference, see [docs/API.md](docs/API.md).
